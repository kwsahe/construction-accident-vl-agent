로컬 에이전트가 mp4에서 대표 프레임을 뽑고 contact sheet를 만든 뒤, VL 모델이 시간순 변화를 비교합니다.

목표는 단순 이미지 분류가 아니라 “사고 전 행동 → 구조물 변화 → 사고 결과”를 JSON으로 구조화하는 것입니다.

1

mp4 입력

2

프레임 추출

overview와 사고 전후 프레임을 시간 라벨과 함께 구성합니다.

3

VL 판단

Qwen2.5-VL이 contact sheet를 보고 사고 순간과 유형을 판단합니다.

4

JSON 검증

primary_type, confidence, details, evidence를 schema 기준으로 확인합니다.

5

DB payload

SPilot ERD의 cctv_events, evidence_photos, tts_alert_logs로 변환합니다.

구현 포인트

사고 판단 모델을 서비스 DB와 바로 섞지 않고, 관찰 → 판단 → 저장 payload를 분리했습니다.

Accident Moment

사고 순간 후보 탐지

`--auto-moment` 옵션으로 전체 영상 overview sheet를 먼저 보고 사고 발생 시점 후보를 찾습니다.

Prompt Design

보이는 사실만 판단

교육 여부, 승인 여부, 사망 여부처럼 영상에서 확인할 수 없는 내용은 단정하지 않도록 프롬프트를 제한했습니다.

Red Zone

위험구역과 사고 판단 분리

Red Zone 진입은 사고 자체가 아니라 사고 경위의 증거 로그 또는 경고 이벤트로 사용합니다.

PT Adapter

YOLO 모델 결합 준비

`.pt` 파일 검증과 선택적 YOLO 추론 결과를 VL 프롬프트 보조 정보로 연결할 수 있게 했습니다.

Fallback

VL 응답 안정화

응답이 JSON이 아니거나 반복 토큰으로 깨지는 경우를 감지해 재시도 또는 fallback 판단을 적용합니다.

실행 산출물과 코드 분리

output, video, model 파일은 Git 추적에서 제외하고 포트폴리오용 구현 사본만 정리했습니다.

프롬프트는 “사고 원인 추정”보다 “시각 근거 기반 판단”에 초점을 맞췄습니다.

특히 이동식 비계 사고에서 낙상과 추락을 혼동하지 않도록, 구조물 상태와 작업자 위치 변화를 시간순으로 보게 했습니다.

VL 판단 스키마 일부
{
  "primary_type": "낙상|추락|화재|기타",
  "secondary_type": "없음|전도|RED_ZONE_ENTRY|기타",
  "confidence": 0.0,
  "timeline": [
    {
      "time": "16s",
      "description": "핵심 장면",
      "structure_state": "정상|이동|기울어짐|전도|불확실"
    }
  ],
  "red_zone_analysis": {
    "entry_detected": true,
    "zone_relation_to_accident": "직접 관련|간접 관련|관련 없음"
  },
  "details": "[사고 경위]\n시간순 원인-결과 흐름"
}
“단순 RED ZONE 진입만으로 사고라고 쓰지 말고, 실제 전도/추락이 보일 때만 사고 발생으로 쓰세요.”

이 제한 덕분에 위험구역 진입 경고와 실제 사고 판단이 분리됩니다. 이후 저장 단계에서는 RED ZONE을 `tts_alert_logs` 또는 사고 경위의 근거로 매핑하고, 사고 유형은 `cctv_events.agent_verdict`에 정규화합니다.

Qwen 출력은 그대로 저장하지 않고 SPilot ERD payload로 변환합니다.

LLM은 판단 JSON을 만들고, 로컬 mapper가 서비스 테이블에 맞는 안정적인 row 형태로 바꿉니다.

primary_type사고 유형을 `worker_fall_from_height`, `worker_slip_and_fall`, `fire_or_smoke` 라벨로 변환
details`cctv_events.agent_summary`에 사고 경위로 저장
confidence`cctv_events.confidence`, `evidence_photos.confidence`에 반영
contact sheet`snapshot_path`, `evidence_photos.photo_url`에 증거 이미지로 연결
red_zone_analysis`tts_alert_logs` 메시지와 사고 경위 근거로 사용
실행 예시
python -m agent.run_remote_judgment \
  --video C:\spilot\backend\agent\video\accident_video.mp4 \
  --auto-moment \
  --pt-model C:\spilot\backend\model\best.pt \
  --run-pt \
  --insert-db

포트폴리오에서 강조할 점

단순 모델 호출보다 “현장 영상 데이터를 서비스 DB로 연결하는 판단 파이프라인”을 설계했다는 점이 핵심입니다.

문제 정의

건설현장 사고 영상을 사람이 매번 확인하지 않아도, 사고 발생 구간과 경위를 구조화합니다.

내 역할

VL 프롬프트 설계, contact sheet 생성 흐름, Qwen 서버 연동, JSON 검증, ERD payload 변환 구현.

확장 방향

YOLO/PT 모델과 tracking을 결합해 작업자 식별과 시간 안정성을 높이고, 이후 RAG 법령 Agent와 연결합니다.
