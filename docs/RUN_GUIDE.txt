기준 경로

1. Conda 가상환경 생성
----------------------
Python 3.11 환경을 권장합니다.

PowerShell 또는 Anaconda Prompt에서 실행:

  cd C:\Users\sangh\Desktop\Code\VL
  conda create -n construction-vl python=3.11 -y
  conda activate construction-vl
  python -m pip install --upgrade pip
  pip install -r requirements.txt


2. 프론트엔드 패키지 설치
------------------------
Node.js가 설치되어 있어야 합니다.

  cd C:\Users\sangh\Desktop\Code\VL
  npm install


3. Colab VL 서버 실행
---------------------
server 폴더 안의 노트북 중 하나를 Colab에 업로드해서 실행합니다.

권장 순서:

  1) server/Qwen3_VL_32B_colab_server.ipynb
     - 메인 추천 모델
     - 사고 원인 분석, 보고서 초안 생성, 긴 영상 reasoning 비교용

  2) server/InternVL3_colab_server.ipynb
     - Qwen3-VL과 perception/reasoning 성능 비교용

  3) server/LLaVA_OneVision_2_8B_colab_server.ipynb
     - 8B급 video baseline
     - 속도/성능 균형 비교용

  4) server/MiniCPM_V_4_5_colab_server.ipynb
     - 경량/효율형 video understanding 후보
     - 빠른 fallback 실험용

  5) server/Judgement_Agent_qwen25_vl_32b_colab_server.ipynb
     - 기존 Qwen2.5-VL 32B baseline

Colab에서 노트북을 모두 실행하면 아래 형태의 주소가 출력됩니다.

  LLM_API_BASE=https://xxxxx.ngrok-free.app/v1

이 주소를 프론트엔드의 "Colab API Base" 입력칸에 넣거나,
agent/.env 파일에 저장해서 사용할 수 있습니다.


4. agent/.env 설정
------------------
agent/.env.example을 참고해서 agent/.env를 만듭니다.

예시:

  LLM_PROVIDER=remote_openai
  LLM_MODEL=Qwen/Qwen3-VL-32B-Instruct
  LLM_API_BASE=https://xxxxx.ngrok-free.app/v1
  LLM_API_KEY=dummy

프론트엔드에서 Colab API Base를 입력하면 해당 값이 우선 사용됩니다.
입력하지 않으면 백엔드가 agent/.env 값을 사용합니다.

11. 자주 발생하는 문제
----------------------
문제: py -3.11 -m venv .venv 실행 시 Python 3.11을 찾지 못함
해결: Anaconda 환경을 사용합니다.

  conda create -n construction-vl python=3.11 -y
  conda activate construction-vl

문제: Colab ngrok 주소가 바뀜
해결: Colab을 다시 실행하면 새 LLM_API_BASE가 출력됩니다. 프론트 입력칸 또는 agent/.env를 새 주소로 바꿉니다.

문제: Qwen3-VL-32B가 Colab에서 OOM 발생
해결: 더 작은 모델 노트북을 사용합니다.

  - LLaVA_OneVision_2_8B_colab_server.ipynb
  - MiniCPM_V_4_5_colab_server.ipynb
  - 기존 Qwen2.5-VL 32B 또는 7B fallback

문제: 프론트엔드에서 백엔드 연결 실패
확인:

  - 백엔드가 127.0.0.1:8000에서 실행 중인지 확인
  - 프론트엔드가 127.0.0.1:5173에서 실행 중인지 확인
  - 브라우저에서 http://127.0.0.1:8000/api/health 접속 확인

13. YOLO pretrained 모델 사용 방법
----------------------------------
사고 영상 데이터가 부족하므로 현재 프로젝트에서는 YOLO를 직접 학습하지 않습니다.
대신 Ultralytics 최신 pretrained 모델을 보조 evidence 생성기로 사용합니다.

2026년 기준 Ultralytics 공식 문서의 최신 계열은 YOLO26입니다.
YOLO26 pretrained 모델은 첫 사용 시 Ultralytics가 자동 다운로드합니다.

설치:

  conda activate construction-vl
  pip install ultralytics

권장 저장 위치:

  C:\Users\sangh\Desktop\Code\VL\agent\models

권장 모델명:

  yolo26n.pt  - 가장 가벼운 detection 모델
  yolo26s.pt  - 정확도와 속도 균형
  yolo26n-pose.pt - pose estimation 실험용

자동 다운로드 예시:

  cd C:\Users\sangh\Desktop\Code\VL\agent\models
  python -c "from ultralytics import YOLO; YOLO('yolo26n.pt')"

프론트엔드 사용:

  1) Analyze 화면에서 "pretrained YOLO evidence 사용" 체크
  2) YOLO 모델 파일에 yolo26n.pt 입력
  3) Colab API Base와 VL 모델 선택
  4) 영상 저장 및 사고 분석 시작

YOLO 역할:

  - 사고를 최종 판단하지 않습니다.
  - 사람 bbox, 객체 위치 변화, frame별 detection을 보조 근거로 생성합니다.
  - 최종 사고 유형과 원인 판단은 VL 모델이 수행합니다.

참고:

  - agent/models 폴더는 Git에 올리지 않습니다.
  - custom 학습한 best.pt가 있다면 agent/models/best.pt에 넣고 프론트의 YOLO 모델 파일에 best.pt를 입력하면 됩니다.
  - 공식 문서: https://docs.ultralytics.com/models/yolo26
